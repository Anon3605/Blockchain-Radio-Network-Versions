01 — Motivation

Amateur radio: critical but vulnerable

Ham radio is the backbone of emergency communication when commercial infrastructure fails. But it operates without encryption or cryptographic authentication — leaving it exposed to interception, spoofing, and injection attacks precisely when reliability matters most.

⬡

No sender authentication

Systems like APRS use simple callsign identification with no cryptographic verification. Any transmitter can impersonate a legitimate station.

⟳

Replay and injection attacks

Recorded packets can be retransmitted without detection. Rogue signals can inject false emergency data into open spectrums.

◈

No tamper evidence

There is no immutable record of transmissions. Messages can be modified in transit with no mechanism to detect alteration.

⊗

Bandwidth constraints

Traditional blockchain consensus is incompatible with AX.25 at 1200 baud — PoW yields 7 TPS with 10KB+ proofs, a fatal mismatch.

The FCC Part 97 framework prohibits encryption in most amateur contexts — yet BERN achieves cryptographic integrity without content encryption, using hash-based authentication that remains legally compliant while fully securing the channel.

02 — Development

Four iterations to real-time security

Each version targeted a specific bottleneck discovered in the previous — from basic hash chaining to full Byzantine Fault Tolerant consensus, without sacrificing sub-millisecond per-hop latency.

Note: All the test results are extracted from docker simulations
Baseline — no authentication

Multi-hop packet relay over Docker

Establishes the producer-repeater-receiver topology over Docker containers with SHA-256 hashing and O(m)-space duplicate detection. ZK proofs are modelled as JSON metadata only — not cryptographically committed. Throughput peaks at 10,000 pkt/s but any node can forge packets undetected.

Session-based — the breakthrough

One-time handshake + HMAC per message

Decouples ZK generation (done once at session start) from per-packet cost. The proof_hash binding anchors every BLAKE2b HMAC tag to the original ZK proof via a 32-byte SHA-256 digest stored in node caches. Voice-grade throughput restored.

Final — distributed trust

Full BFT consensus — no single point of trust

Replaces the central Producer with Byzantine Fault Tolerant majority-vote consensus (f = ⌊(n−1)/2⌋ faults tolerated). Control plane on port 7000 handles async blockchain ops; data plane on ports 5000+ carries live traffic at 0.1–0.15ms per hop. Disk-hybrid L0/L1 caching separates proof-hash from full session state.

03 — Design

Dual-plane architecture

The key innovation is separating trust establishment from message delivery. The control plane negotiates consensus asynchronously; the data plane forwards packets in real time with cached session credentials.
